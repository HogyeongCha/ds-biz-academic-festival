# Step 4: 여정 프로파일링

## 목적

AIDA 퍼널 기반 상태 전이 모델을 구현하여 고객의 현재 구매 여정 단계를 추론하고, 추천 타이밍을 동적으로 조정하기 위한 여정 프로파일을 구축한다. 기존 추천 시스템이 간과하기 쉬운 '고객의 의도'와 '니즈 변화'를 포착하여, 단순히 무엇을 추천할지가 아니라 언제 추천할지를 결정하는 맥락적 개인화를 실현하는 것이 이 단계의 핵심 목표이다. 이를 통해 각 사용자의 행동 로그를 분석하여 현재 여정 단계를 분류하고, 여정 단계별로 차별적인 추천 전략(예: Awareness 단계에서는 Content 비중 증가, Action 단계에서는 CF 비중 증가)을 수립할 수 있는 기반 데이터를 생성한다.

## 방법론

> **참조: Paper §4.3 상호작용 (여정) 프로파일링**

본 연구에서는 마케팅 분야에서 널리 활용되는 AIDA(Attention, Interest, Desire, Action) 모델을 기반으로 고객의 구매 여정 단계를 정의하고, 이를 상태 전이(State Transition) 모델로 구현하였다. 각 사용자의 행동 로그를 분석하여 현재 여정 단계를 실시간으로 분류하며, 이 여정 상태는 추천 시스템에서 동적 가중치 조절에 활용된다.

### 처리 절차

**1단계: 데이터 전처리**

원본 상호작용 데이터(`combined_interactions.csv`)에서 `timestamp` 컬럼을 날짜/시간 형식으로 파싱하고, 유효하지 않은 타임스탬프를 포함하는 행은 분석에서 제외하여 데이터의 정합성을 확보한다. 상호작용 유형(`interaction_type`)은 소문자로 정규화하여 일관된 분류 기준을 적용한다.

**2단계: 사용자별 최근 상호작용 식별**

각 `user_id`를 기준으로 데이터를 시간순 정렬한 후, 가장 최근의 상호작용 기록(마지막 활동 시점, 유형, 관련 상품)을 추출한다. 이 정보는 고객의 현재 상태를 파악하는 데 가장 중요한 기준점이 된다.

**3단계: 30분 세션 기반 행동 지표 산출**

각 사용자의 마지막 상호작용 시점을 기준으로, 해당 시점으로부터 30분 이내의 활동을 '세션'으로 정의한다. 이 세션 내에서 발생한 상호작용 유형별 횟수를 계산한다:
- `session_view_count`: 세션 내 View 횟수
- `session_like_count`: 세션 내 Like(Cart/Wishlist) 횟수
- `session_purchase_count`: 세션 내 Purchase 횟수

**4단계: AIDA 기반 여정 상태 분류**

세션 행동 지표와 마지막 상호작용 유형을 복합적으로 고려하여, 다음 6단계 + 1(Unknown) 중 하나로 분류한다:

| 여정 단계 | 전이 조건 | 마케팅적 시사점 |
|-----------|-----------|----------------|
| **Dormant** (휴면) | 마지막 상호작용으로부터 60일 이상 경과 | 재활성화 캠페인 필요 |
| **Awareness** (인지) | 마지막 행동이 View이며, 세션 내 Like·Purchase 모두 0건 | 상품 인지 단계 — Content 추천 비중 확대 |
| **Interest** (관심) | 마지막 행동이 Like이고, Purchase 없음 | 관심 표현 단계 — 유사 상품 추천 강화 |
| **Desire** (욕구) | 마지막 행동이 Like이고, 세션 내 View 3회 이상 | 구매 욕구 형성 — 프로모션/할인 추천 |
| **Action** (행동) | 세션 내 Purchase 발생 (마지막 행동이 Purchase가 아닌 경우) | 구매 행동 단계 — CF 비중 확대 |
| **Post-Purchase** (구매 후) | 마지막 행동이 Purchase | 구매 후 단계 — 교차판매/리뷰 유도 |
| **Unknown** | 상기 조건에 모두 해당하지 않는 경우 | 추가 데이터 확보 필요 |

**5단계: Journey Profile 생성**

위 과정을 통해 각 사용자별로 `user_id`, `last_interacted_product_id`, `last_interaction_type`, `last_interaction_timestamp`, 세션 행동 지표(`session_view_count`, `session_like_count`, `session_purchase_count`), 그리고 최종 판별된 `current_journey_stage`를 포함하는 여정 프로파일을 생성한다.

이 여정 상태는 후속 추천 단계(Step 5)에서 동적 가중치 조절에 활용된다. 예를 들어 Awareness 단계의 고객에게는 Content 기반 추천 비중을 높이고, Action 단계의 고객에게는 CF 기반 추천 비중을 높이는 등 여정 맥락에 맞는 차별적 추천 전략을 수립할 수 있다.

## 결과 요약

### 여정 단계별 사용자 분포

전체 2,999명의 사용자에 대해 AIDA 기반 여정 단계를 분류한 결과:

| 여정 단계 | 사용자 수 | 비율 |
|-----------|-----------|------|
| **Dormant** (휴면) | 2,099명 | 70.0% |
| **Interest** (관심) | 386명 | 12.9% |
| **Awareness** (인지) | 266명 | 8.9% |
| **Post-Purchase** (구매 후) | 248명 | 8.3% |
| **Desire** (욕구) | 0명 | 0.0% |
| **Action** (행동) | 0명 | 0.0% |

- Dormant(휴면) 고객이 전체의 70.0%로 압도적 다수를 차지하며, 이는 데이터셋의 상호작용 기간과 기준일 간의 시간 차이에 기인한다.
- 활성 사용자 중에서는 Interest(12.9%) > Awareness(8.9%) > Post-Purchase(8.3%) 순으로 분포하며, 관심 표현 단계의 고객이 가장 많은 비중을 차지한다.
- Desire와 Action 단계의 사용자가 0명인 것은, 해당 조건(Like + View 3회 이상, 세션 내 Purchase 발생 후 추가 활동)을 충족하는 사용자가 데이터셋에 존재하지 않았음을 의미한다.

## 회고

### 왜 이 방법을 선택했는가

고객의 구매 여정을 모델링하기 위한 프레임워크로 AIDA 모델을 선택한 이유는 세 가지였다. 첫째, AIDA 모델은 소비자의 의사결정 과정을 '주의 → 관심 → 욕구 → 행동'이라는 직관적이고 체계적인 단계로 설명하여, 마케팅 실무자와 데이터 분석가 모두가 동일한 관점에서 고객 여정을 이해할 수 있었다. 둘째, 각 단계가 마케팅 전략과 직접 연결 가능하여, 여정 단계별로 차별적인 추천 전략(Content 비중 조절, CF 비중 조절 등)을 자연스럽게 도출할 수 있었다. 셋째, 다른 소비자 행동 모델(Hierarchy of Effects, SOR 모델 등)과 비교했을 때, 인지적 여정을 추적하는 데 있어 AIDA가 가장 적절한 틀로 평가받고 있었다(Lavidge & Steiner, 1961).

상태 전이 모델로 구현한 것은, 고객의 여정이 정적인 분류가 아니라 시간에 따라 변화하는 동적 프로세스라는 점을 반영하기 위함이었다. 30분 세션 기반 접근은 고객의 '현재' 행동 맥락을 포착하는 데 적합한 시간 단위로 판단하였다.

### 잘된 점

- **30분 세션 기반 접근으로 실시간 행동 맥락 반영**: 단순히 전체 이력을 집계하는 것이 아니라, 마지막 상호작용 시점 기준 30분 이내의 활동을 세션으로 정의함으로써 고객의 '현재' 행동 맥락을 효과적으로 포착할 수 있었다. 이는 추천 타이밍의 적시성을 높이는 데 핵심적인 역할을 하였다.
- **여정 단계별 차별적 추천 전략 수립 가능**: AIDA 단계를 명확하게 정의함으로써, 각 단계에 맞는 추천 전략(Awareness → Content 비중 확대, Action → CF 비중 확대 등)을 체계적으로 설계할 수 있는 기반을 마련하였다. 이는 후속 Step 5의 동적 가중치 결합에서 직접적으로 활용되었다.
- **마케팅 프레임워크와 데이터 분석의 자연스러운 결합**: AIDA라는 마케팅 이론을 상태 전이 모델이라는 데이터 분석 기법으로 구현함으로써, 이론과 실무의 간극을 좁히는 접근을 실현하였다.

### 개선할 점

- **30분 세션 임계값의 최적화 실험 부족**: 30분이라는 세션 임계값은 일반적인 이커머스 세션 정의에 기반한 것이지만, 본 데이터셋에 최적화된 값인지에 대한 체계적인 실험이 부족하였다. 15분, 45분, 60분 등 다양한 임계값에 대한 비교 실험을 수행했다면, 데이터 특성에 더 적합한 세션 정의를 도출할 수 있었을 것이다.
- **여정 전이 확률 모델링 미적용**: 현재 구현은 각 사용자의 '현재' 상태만을 판별하는 스냅샷 방식이다. 마르코프 체인 등을 활용하여 여정 단계 간 전이 확률을 모델링했다면, 고객이 다음에 어떤 단계로 이동할지를 예측하여 선제적 추천이 가능했을 것이다.
- **Dormant 비율의 높은 편중**: 전체 사용자의 70%가 Dormant로 분류되어, 활성 사용자에 대한 여정 분류의 세분화 효과가 제한적이었다. Dormant 임계값(60일)의 조정이나, Dormant 내에서의 추가 세분화(예: 최근 Dormant vs. 장기 Dormant)를 고려할 필요가 있다.

### 배운 점

- **마케팅 프레임워크(AIDA)와 데이터 분석의 결합 가능성**: AIDA라는 고전적 마케팅 모델이 데이터 기반 추천 시스템에서도 유효한 분류 체계로 작동할 수 있음을 확인하였다. 마케팅 이론이 단순한 개념적 도구를 넘어, 실제 데이터 파이프라인의 분류 로직으로 구현될 수 있다는 점은 이론과 실무의 융합 가능성을 보여주었다.
- **추천 타이밍의 중요성**: '무엇을 추천할 것인가'만큼이나 '언제 추천할 것인가'가 중요하다는 점을 체감하였다. 동일한 상품이라도 고객의 여정 단계에 따라 추천의 효과가 크게 달라질 수 있으며, 이를 위해 고객의 현재 맥락을 실시간으로 파악하는 여정 프로파일링이 필수적이라는 인사이트를 얻었다.
