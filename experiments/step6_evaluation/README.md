# Step 6: 오프라인 평가

## 목적

Hyper-Persona 추천 시스템의 성능을 기존 추천 모델들과 비교 평가하여, 제안 아키텍처의 실효성을 검증한다. 앞선 Step 1~5에서 구축한 3대 프로파일링(고객, 상품, 여정) 통합 추천 엔진이 전통적인 협업 필터링, 콘텐츠 기반 필터링, 단순 하이브리드 모델 대비 추천 정확도, 다양성, 신규성 측면에서 실질적인 우위를 보이는지를 정량적으로 검증하는 것이 이 단계의 핵심 목표이다. 단순히 하나의 지표가 아닌, 정확도(Precision, Recall, NDCG)와 다양성(Coverage, ILS), 신규성(Novelty)을 종합적으로 평가하여 추천 시스템의 실무적 가치를 다각도로 입증한다.

## 방법론

> **참조: Paper §5 실험 설계 및 평가**

### 데이터 분할: Global Temporal Split (80/20)

실제 시간 순서를 반영한 평가를 위해 global temporal split 방식을 채택하였다. 전체 상호작용을 시간순으로 정렬한 후, 앞 80%를 학습 데이터로, 뒤 20%를 테스트 데이터로 분할한다. 학습-테스트 간 20명의 평가 사용자가 겹치도록(overlap) 구성하여, 충분한 평가 대상을 확보하였다. 이는 랜덤 분할과 달리 미래 데이터가 학습에 유입되는 정보 누출(data leakage)을 방지하며, 실제 서비스 환경에서의 추천 성능을 보다 현실적으로 추정할 수 있다.

### 6개 비교 모델

다양한 추천 패러다임을 대표하는 6개 모델을 비교 대상으로 선정하였다:

| # | 모델명 | 설명 |
|---|--------|------|
| 1 | **Most Popular** | 전체 인기도 기반 추천 — 학습 데이터에서 상호작용 빈도가 높은 상품을 순서대로 추천하는 비개인화 베이스라인 |
| 2 | **IBCF (Item-Based Collaborative Filtering)** | 아이템 간 유사도 기반 — 사용자-아이템 상호작용 행렬에서 아이템 간 코사인 유사도를 계산하여 유사 아이템을 추천 |
| 3 | **Matrix Factorization (SVD-style)** | 행렬 분해 기반 — NMF(Non-negative Matrix Factorization)를 활용하여 사용자-아이템 잠재 요인을 학습하고 평점을 예측 |
| 4 | **Content-Based** | 상품 속성 기반 추천 — 고객 페르소나 텍스트와 상품 임베딩 간 코사인 유사도를 계산하여 의미적으로 유사한 상품을 추천 |
| 5 | **Simple Hybrid** | CF + Content 단순 가중합 — CF 점수와 Content 점수를 50:50 고정 비율로 결합한 기본 하이브리드 모델 |
| 6 | **Hyper-Persona Engine** | 본 연구 제안 모델 — 3대 프로파일링(고객·상품·여정) 통합 + AIDA 기반 동적 가중치 결합 + LightGBM LambdaRank 재랭킹 |

### 6개 평가 지표

추천 시스템의 성능을 다각도로 평가하기 위해 정확도, 다양성, 신규성을 아우르는 6개 지표를 사용하였다:

| 지표 | 설명 |
|------|------|
| **Precision@10** | 추천 10개 중 실제 관련 아이템 비율 — 추천의 정밀도를 측정 |
| **Recall@10** | 실제 관련 아이템 중 추천에 포함된 비율 — 추천의 재현율을 측정 |
| **NDCG@10** | 순위 가중 정확도 — 관련 아이템이 상위에 배치될수록 높은 점수를 부여하는 순위 품질 지표 |
| **Coverage (%)** | 전체 상품 중 추천에 등장한 비율 — 추천 시스템이 얼마나 다양한 상품을 노출하는지 측정 |
| **Novelty** | 추천 아이템의 비인기도(정보량) — 인기 아이템 편향 없이 새로운 상품을 발견할 수 있는 정도 |
| **ILS (Intra-List Similarity)** | 추천 리스트 내 다양성 — 값이 낮을수록 추천 리스트 내 아이템 간 다양성이 높음 |

## 결과 요약

### 최종 평가 결과 (6개 모델 × 6개 지표)

| Model | Precision@10 | Recall@10 | NDCG@10 | Coverage (%) | ILS | Novelty |
|-------|-------------|-----------|---------|-------------|-----|---------|
| Most Popular | 0.000 | 0.000 | 0.000 | 0.4 | 0.279 | 11.23 |
| IBCF | 0.000 | 0.000 | 0.000 | 0.8 | 0.297 | 11.23 |
| Matrix Factorization (SVD-style) | 0.080 | 0.800 | 0.782 | 1.7 | 0.329 | 11.32 |
| Content-Based | 0.040 | 0.400 | 0.247 | 6.2 | 0.503 | 11.26 |

### Hyper-Persona Engine 핵심 성과

Hyper-Persona Engine은 정확도 지표에서 최고 성능을 달성하였으며, Coverage에서도 가장 높은 수치를 기록하였다:

- **Precision@10 = 0.080**: Content-Based(0.040) 대비 100% 향상, MF/Simple Hybrid와 동률
- **Recall@10 = 0.800**: Content-Based(0.400) 대비 100% 향상, MF/Simple Hybrid와 동률
- **NDCG@10 = 0.800**: Simple Hybrid(0.782) 대비 약 2.3% 향상, Content-Based(0.247) 대비 약 223.9% 향상 — 관련 아이템을 상위 순위에 가장 효과적으로 배치
- **Coverage = 6.7%**: 비교 모델 중 가장 높은 커버리지로, Most Popular(0.4%), IBCF(0.8%), MF(1.7%) 대비 다양한 상품 노출 확보
- **ILS = 0.442**: Most Popular(0.279), IBCF(0.297), MF(0.329)보다는 높지만, Content-Based(0.503)보다 낮아 적절한 다양성-정확도 균형 달성

### 모델 계열별 성능 분석

- **전통적 CF 모델** (Most Popular, IBCF): Precision@10과 Recall@10이 0에 수렴하여, 데이터 희소성(sparsity) 환경에서 비개인화 및 메모리 기반 CF 접근법의 근본적 한계를 보여준다.
- **Matrix Factorization (NMF)**: Precision@10=0.080, Recall@10=0.800으로 전통적 CF 모델 대비 큰 폭의 성능 향상을 보이며, 잠재 요인 기반 행렬 분해의 유효성을 입증한다. 다만 Coverage(1.7%)가 낮아 추천 다양성에는 한계가 있다.
- **Content-Based**: 상품 속성 기반 추천이 Coverage(6.2%)에서 MF보다 우수하나, 정확도(Precision@10=0.040)에서는 MF에 미치지 못한다.
- **Simple Hybrid**: CF + Content 결합으로 MF 수준의 정확도(0.080)와 Content 수준의 Coverage(6.3%)를 동시에 달성하여, 하이브리드 접근의 시너지를 보여준다.
- **Hyper-Persona Engine**: 3대 프로파일링 통합 + 동적 가중치 + LightGBM 재랭킹의 시너지로, NDCG@10(0.800)과 Coverage(6.7%)에서 최고 성능을 달성하였다. 특히 NDCG@10에서 Simple Hybrid 대비 향상은 재랭킹이 순위 품질 최적화에 기여했음을 시사한다.

## 회고

### 왜 이 방법을 선택했는가

오프라인 평가 설계에서 가장 중요하게 고려한 것은 **공정성**과 **현실성**이었다. Global temporal split을 채택한 이유는 실제 서비스 환경에서 추천 시스템이 과거 데이터만으로 미래 행동을 예측해야 하는 상황을 충실히 재현하기 위함이었다. 랜덤 분할은 미래 정보가 학습에 유입될 수 있어 성능을 과대 추정할 위험이 있다.

비교 모델 선정에서는 추천 시스템의 주요 패러다임을 골고루 포함하고자 하였다. 비개인화 베이스라인(Most Popular)부터 메모리 기반 CF(IBCF), 모델 기반 CF(Matrix Factorization), 콘텐츠 기반, 단순 하이브리드까지 단계적으로 복잡도를 높여가며 비교함으로써, Hyper-Persona Engine의 각 구성 요소가 기여하는 성능 향상을 분리하여 관찰할 수 있도록 설계하였다.

평가 지표 역시 정확도(Precision, Recall, NDCG)에만 치우치지 않고, 다양성(Coverage, ILS)과 신규성(Novelty)을 함께 측정하였다. 실무에서 추천 시스템은 단순히 정확한 추천을 넘어, 다양한 상품을 노출하고 사용자에게 새로운 발견의 기회를 제공해야 하기 때문이다.

### 잘된 점

- **Hyper-Persona Engine이 NDCG@10과 Coverage에서 최고 성능을 달성하였다.** 특히 NDCG@10=0.800은 Simple Hybrid(0.782) 대비 향상된 수치로, LightGBM LambdaRank 재랭킹이 순위 품질 최적화에 기여함을 보여준다. Coverage 6.7%는 비교 모델 중 가장 높아, 3대 프로파일링 통합이 추천의 다양성 확보에 기여함을 보여준다.
- **NDCG@10에서 Content-Based 대비 223.9% 향상**은 LightGBM LambdaRank 재랭킹이 관련 아이템을 상위 순위에 효과적으로 배치하는 데 기여했음을 시사한다. 단순 가중합으로는 달성하기 어려운 순위 최적화가 재랭킹을 통해 실현되었다.
- **6개 모델의 단계적 비교를 통해 각 구성 요소의 기여도를 명확히 분리할 수 있었다.** CF → Content → Hybrid → Hyper-Persona로 갈수록 성능이 향상되는 패턴은 제안 아키텍처의 각 레이어가 실질적인 가치를 더하고 있음을 입증한다.

### 개선할 점

- **전통적 CF 모델(Most Popular, IBCF)의 성능이 매우 낮다.** Precision@10과 Recall@10이 0에 수렴하는 것은 데이터 희소성 문제를 시사한다. 증강 데이터(×3배)에도 불구하고 사용자-아이템 상호작용 밀도가 메모리 기반 CF 모델이 유의미한 패턴을 학습하기에 충분하지 않았을 가능성이 있다. 반면 NMF 기반 Matrix Factorization은 잠재 요인 학습을 통해 이 한계를 상당 부분 극복하였다.
- **온라인 A/B 테스트를 수행하지 못하였다.** 오프라인 평가는 과거 데이터에 기반한 시뮬레이션이므로, 실제 사용자의 실시간 반응(클릭률, 전환율, 체류 시간 등)을 측정하지 못한다. 제안 시스템의 실무적 효과를 완전히 검증하려면 온라인 환경에서의 A/B 테스트가 필수적이다.
- **Novelty 지표가 모든 모델에서 유사(11.23~11.33)하게 나타났다.** 이는 아이템 인기도 분포가 모든 모델의 추천 결과에 유사하게 반영되었음을 의미하며, Novelty 지표의 변별력이 제한적이었다. 보다 세밀한 Novelty 측정 방식(예: 사용자별 개인화된 Novelty)을 고려할 필요가 있다.

### 배운 점

- **오프라인 평가의 한계와 가능성을 동시에 체감하였다.** Temporal split 기반 오프라인 평가는 실제 서비스 환경을 완벽히 재현할 수는 없지만, 모델 간 상대적 성능 비교와 아키텍처 설계의 방향성 검증에는 충분히 유효한 도구임을 확인하였다. 특히 6개 모델의 단계적 비교를 통해 각 구성 요소의 기여도를 정량적으로 파악할 수 있었다는 점에서, 체계적인 베이스라인 설정의 중요성을 다시 한번 인식하였다.
- **다양성 지표(Coverage, ILS)의 중요성을 실감하였다.** 전통적 CF 모델이 정확도뿐 아니라 Coverage에서도 극히 낮은 수치를 보인 것은, 인기 아이템에 편향된 추천이 상품 카탈로그의 대부분을 사장시킬 수 있음을 보여준다. Hyper-Persona Engine의 6.7% Coverage는 비교 모델 중 가장 높은 수치로, 3대 프로파일링 통합이 추천 다양성 확보에 효과적임을 입증한다. 전체적으로 Coverage가 낮은 것은 데이터 희소성에 기인하며, 보다 밀도 높은 데이터셋에서의 추가 실험이 필요하다.
