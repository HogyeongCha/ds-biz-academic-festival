# Step 5: 하이브리드 추천·재랭킹

## 목적

3대 프로파일링(고객, 상품, 여정)을 통합하여 하이브리드 추천 후보군을 생성하고, LightGBM LambdaRank 기반 전략적 재랭킹을 수행하여 개인화된 추천 리스트를 생성한다. 이 단계는 앞선 Step 2~4에서 구축한 고객 페르소나, 상품 임베딩, 여정 프로파일을 하나의 추천 파이프라인으로 결합하는 핵심 단계이다. 단순히 '어떤 상품을 추천할 것인가'를 넘어, '누구에게(고객 프로파일)', '무엇을(상품 프로파일)', '언제(여정 프로파일)' 추천할 것인가를 통합적으로 고려하는 Hyper-Persona 추천 엔진의 구현이 이 단계의 핵심 목표이다.

## 방법론

> **참조: Paper §4.4 통합 추천 파이프라인 및 §5.2.2 통합 추천 파이프라인 구현**

본 연구의 통합 추천 파이프라인은 **하이브리드 후보군 생성 → LightGBM LambdaRank 재랭킹 → 최종 추천 리스트 생성**의 3단계로 구성된다.

### 1단계: 하이브리드 후보군 생성

두 가지 독립적인 추천 후보군을 병렬로 생성한 후, AIDA 기반 동적 가중치로 결합한다.

**CF(Collaborative Filtering) 후보 생성**

사용자-아이템 상호작용 행렬을 기반으로 협업 필터링 후보를 생성한다. 상호작용 유형별 암묵적 평점을 부여하여(View=1.0, Like=3.0, Purchase=5.0) 사용자-아이템 평점 행렬을 구성한다.
- **NMF(Non-negative Matrix Factorization) 사용**: 잠재 요인 수 20, 최대 반복 400회로 설정하여 사용자-아이템 잠재 요인을 학습한다. 비음수 제약 조건으로 인해 해석 가능성이 높고, 희소 행렬에서도 안정적으로 동작한다. (Surprise SVD는 numpy 2.x 호환성 문제로 fallback)
- **SVD fallback 불가**: `PAPER_REPRO=1` 모드에서는 Surprise 라이브러리의 SVD 알고리즘을 시도하나, numpy 2.x 환경에서 호환성 문제(`np.float_` 제거 등)로 인해 NMF로 자동 fallback된다. 따라서 현재 실행 환경에서는 NMF가 실질적인 CF 모델로 사용된다.

**시맨틱 유사도 후보 생성**

고객의 페르소나 정보(Step 2)와 상품의 의미적 임베딩(Step 3)을 활용하여 Content-Based 후보를 생성한다. 고객별로 페르소나 문장, 관심사 키워드, 구매 동기, 주요 구매 카테고리, 여정 단계를 `[SEP]` 구분자로 결합한 사용자 텍스트를 구성하고, 상품의 통합 텍스트(combined_text + attribute_tags + cluster_summary)와의 코사인 유사도를 계산한다.
- **SBERT 임베딩**: `PAPER_REPRO=1` 모드에서는 `paraphrase-mpnet-base-v2` 모델로 768차원 벡터를 생성하여 의미적 유사도를 정밀하게 측정한다.
- **TF-IDF fallback**: 빠른 실행 모드에서는 TF-IDF(max_features=2048) 기반 벡터화로 대체하여 실행 속도를 확보한다.

**AIDA 기반 동적 가중치 결합**

여정 단계(Step 4)와 고객의 관여도(involvement_diversity), 행동 클러스터 레이블에 따라 CF와 Content의 비율을 동적으로 조절한다. 이는 단순 가중합 하이브리드의 한계를 극복하고, 마케팅 전략을 추천 로직에 내재화하기 위한 핵심 설계이다.

| 고객 유형 | CF 비중 | Content 비중 | 전략적 근거 |
|-----------|---------|-------------|------------|
| **고관여 고객** (관여도 ≥ 중앙값 또는 VIP 탐색가/신규 탐색 고객) | 40% | 60% | 탐색 성향이 강한 고객에게 다양한 상품을 의미적으로 매칭하여 노출 |
| **저관여 고객** (관여도 < 중앙값) | 70% | 30% | 구매 패턴이 명확한 고객에게 협업 필터링 기반의 검증된 추천 제공 |

각 후보군에서 상위 100~120개 아이템을 추출한 후, 이미 구매한 상품을 제외하고 중복 없이 통합하여 최종 후보 풀을 구성한다.

### 2단계: LightGBM LambdaRank 재랭킹

후보군에 대해 LightGBM의 LambdaRank 알고리즘으로 전략적 재랭킹을 수행한다. 단순 가중합 점수를 넘어, 다차원적 피처를 학습하여 순위를 최적화한다.

**입력 피처 구성**

| 피처 카테고리 | 포함 피처 |
|--------------|----------|
| 모델 점수 | CF 점수, Content(LLM) 점수, 하이브리드 점수, CF/Content 가중치 |
| 고객 피처 | 행동 클러스터, 관여도(involvement_diversity), 연령대, 성별 |
| 여정 피처 | AIDA 여정 단계 (인코딩), 행동 클러스터 레이블 (인코딩) |
| 상품 피처 | 가격, 인기도, 주요 클러스터, 서브 클러스터 |
| 교차 피처 | 선호 클러스터 일치 여부, 가격 민감도 매칭, 키워드 오버랩 수 |

**LightGBM 하이퍼파라미터**

- `objective`: lambdarank
- `metric`: ndcg
- `n_estimators`: 200
- `num_leaves`: 31
- `learning_rate`: 0.05
- `subsample`: 0.8, `colsample_bytree`: 0.8
- `random_state`: 42

**적용 조건**: 학습 데이터에 양성 레이블(실제 상호작용이 존재하는 사용자-아이템 쌍)이 충분한 경우에만 LightGBM 재랭킹을 적용한다. 양성 레이블이 부족한 경우(기본 임계값: 100건 이상, 양성 비율 1% 이상), 1단계에서 산출한 하이브리드 가중합 점수를 그대로 사용하는 fallback 전략을 적용한다.

### 3단계: 최종 추천 리스트 생성

재랭킹 점수 기준으로 사용자별 상위 10개 아이템을 선정하여 최종 추천 리스트를 생성한다. 각 추천 항목에는 CF 점수, Content 점수, 하이브리드 점수, 재랭킹 점수, 여정 단계, 행동 클러스터, 상품 속성 태그 등 추천 사유를 추론할 수 있는 메타데이터가 포함된다.

## 실행 방법

### 파이프라인 전체 경로에서 실행

```bash
python pipeline/scripts/phase5_recommendation.py
```

### Step 내 독립 실행

```bash
python code/phase5_recommendation.py
```

### 환경변수 옵션

| 환경변수 | 설명 | 예시 |
|----------|------|------|
| `FAST_MODE` | 빠른 실행 모드 (후보군 축소) | `FAST_MODE=1` |
| `PAPER_REPRO` | 논문 재현 모드 (Surprise SVD 시도 → numpy 2.x 문제로 NMF fallback + SBERT 사용) | `PAPER_REPRO=1` |
| `USE_GEMINI` | Gemini LLM 활용 여부 | `USE_GEMINI=1` |
| `EMBEDDING_BACKEND` | 임베딩 백엔드 선택 (`sbert` / `tfidf`) | `EMBEDDING_BACKEND=tfidf` |
| `CANDIDATE_TOPK` | 후보군 크기 (기본: FAST=100, 일반=120) | `CANDIDATE_TOPK=120` |
| `RECO_TOPN` | 사용자별 최종 추천 수 (기본: 10) | `RECO_TOPN=10` |

```bash
# 기본 실행 (NMF + TF-IDF, 빠른 모드)
FAST_MODE=1 python code/phase5_recommendation.py

# 논문 재현 모드 (NMF + SBERT + Gemini; SVD는 numpy 2.x 호환성 문제로 NMF fallback)
PAPER_REPRO=1 USE_GEMINI=1 python code/phase5_recommendation.py
```

> **참고**: 이 단계는 Step 1~4의 산출물(`combined_interactions.csv`, `customer_profiles.csv`, `product_profiles.csv`, `journey_profiles.csv`)을 입력으로 사용한다. 모든 이전 단계가 먼저 실행되어 해당 파일들이 존재해야 한다.

## 결과 요약

### 동적 가중치 전략 (여정 단계별 CF/Content 비율 조절)

고객의 관여도와 행동 클러스터에 따라 CF와 Content 추천의 비율을 동적으로 조절하는 전략을 적용하였다. 이는 AIDA 여정 모델의 마케팅적 시사점을 추천 로직에 직접 반영한 것이다.

| 여정 맥락 | CF 비중 | Content 비중 | 마케팅 전략 |
|-----------|---------|-------------|------------|
| Awareness (인지) — 탐색 단계 | 낮음 (40%) | 높음 (60%) | 다양한 상품 노출로 탐색 촉진 |
| Interest/Desire (관심/욕구) | 균형 | 균형 | CF와 Content의 시너지 활용 |
| Action/Post-Purchase (행동/구매 후) | 높음 (70%) | 낮음 (30%) | 구매 패턴 기반 검증된 추천 |

실제 구현에서는 관여도(involvement_diversity)의 중앙값과 행동 클러스터 레이블(VIP 탐색가, 신규 탐색 고객 등)을 기준으로 고관여/저관여를 판별하여 가중치를 결정한다.

### LightGBM 재랭킹 적용 조건 및 효과

- **적용 조건**: 양성 레이블(실제 상호작용 존재) 100건 이상 + 양성 비율 1% 이상일 때 LightGBM LambdaRank 적용
- **미충족 시**: 하이브리드 가중합 점수(cf_weight × cf_score + llm_weight × llm_score)를 그대로 사용
- **재랭킹 효과**: 단순 가중합으로는 포착할 수 없는 피처 간 비선형 상호작용(예: 선호 클러스터 일치 + 가격 민감도 매칭)을 학습하여, 추천의 정밀도와 전략적 적합성을 향상시킨다

## 회고

### 왜 이 방법을 선택했는가

단순 가중합 하이브리드 추천의 한계를 극복하기 위해 두 가지 핵심 설계를 도입하였다. 첫째, AIDA 기반 동적 가중치를 통해 마케팅 전략을 추천 로직에 내재화하였다. 기존 하이브리드 추천 시스템은 CF와 Content의 비율을 고정(예: 5:5)하여 모든 고객에게 동일한 전략을 적용하는 한계가 있었다. 본 연구에서는 고객의 여정 단계와 관여도에 따라 이 비율을 동적으로 조절함으로써, 탐색 단계의 고객에게는 다양한 상품을 노출하고, 구매 단계의 고객에게는 검증된 추천을 제공하는 차별적 전략을 구현하였다. 둘째, LightGBM LambdaRank를 활용한 전략적 재랭킹을 도입하였다. 단순 점수 합산이 아닌, 고객 피처·상품 피처·여정 피처·교차 피처를 종합적으로 학습하여 순위를 최적화함으로써, 추천의 정밀도를 한 단계 끌어올리고자 하였다.

### 잘된 점

- **3대 프로파일링의 유기적 통합**: 고객 프로파일(Step 2), 상품 프로파일(Step 3), 여정 프로파일(Step 4)이 추천 파이프라인에서 각각 CF 점수, Content 점수, 동적 가중치로 자연스럽게 결합되었다. 각 프로파일링 단계가 독립적으로 가치를 가지면서도, 통합 시 시너지를 발휘하는 구조를 실현하였다.
- **여정 맥락을 반영한 동적 가중치**: 고객의 현재 여정 단계에 따라 추천 전략을 자동으로 조절하는 메커니즘을 구현함으로써, '언제 무엇을 추천할 것인가'라는 마케팅적 질문에 데이터 기반으로 답할 수 있는 체계를 마련하였다.
- **다단계 fallback 전략**: NMF → SVD, SBERT → TF-IDF, LightGBM → HistGradientBoosting → 하이브리드 점수 등 각 단계에서 실패 시 대안을 제공하는 견고한 파이프라인을 구축하여, 다양한 실행 환경에서도 안정적으로 동작하도록 설계하였다.

### 개선할 점

- **NMF/SVD 외 딥러닝 CF 모델 미적용**: 현재 CF 모델은 전통적인 행렬 분해(NMF, SVD)에 한정되어 있다. Neural Collaborative Filtering(NCF)이나 Variational Autoencoder(VAE) 기반 CF 모델을 적용했다면, 사용자-아이템 상호작용의 비선형 패턴을 더 효과적으로 포착할 수 있었을 것이다.
- **재랭킹 피처 엔지니어링 부족**: 현재 피처 세트는 기본적인 모델 점수, 고객/상품 속성, 교차 피처로 구성되어 있다. 시간적 패턴(구매 주기, 시즌성), 세션 내 행동 시퀀스, 상품 간 보완재/대체재 관계 등 보다 정교한 피처를 추가했다면 재랭킹의 효과를 더 높일 수 있었을 것이다.
- **동적 가중치의 세분화 부족**: 현재 고관여/저관여 이분법적 분류에 기반한 가중치 조절은 다소 단순하다. 여정 단계별로 더 세밀한 가중치 테이블을 설계하거나, 가중치 자체를 학습 가능한 파라미터로 설정하는 접근이 필요하다.

### 배운 점

- **추천 시스템에서 마케팅 전략 내재화의 가치**: AIDA 기반 동적 가중치를 통해 마케팅 프레임워크를 추천 알고리즘에 직접 반영할 수 있음을 확인하였다. 이는 추천 시스템을 단순한 예측 도구가 아닌, 마케팅 의사결정 지원 시스템으로 진화시킬 수 있는 가능성을 보여주었다. 기술적 성능 지표(Precision, Recall)만을 추구하는 것이 아니라, 비즈니스 맥락과 고객 여정을 고려한 추천이 실무적으로 더 큰 가치를 가질 수 있다는 인사이트를 얻었다.
- **재랭킹의 실질적 효과**: LightGBM LambdaRank를 통한 재랭킹이 단순 가중합 대비 추천 품질을 향상시킬 수 있음을 확인하였다. 특히 교차 피처(선호 클러스터 일치, 가격 민감도 매칭)가 재랭킹에서 중요한 역할을 하며, 이는 고객과 상품 간의 다차원적 적합성을 포착하는 데 기여하였다. 재랭킹은 후보군 생성 단계에서 놓칠 수 있는 세밀한 선호를 보정하는 효과적인 메커니즘임을 체감하였다.
