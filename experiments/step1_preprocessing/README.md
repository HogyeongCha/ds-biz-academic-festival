# Step 1: 데이터 전처리 및 증강

## 목적

원시 이커머스 데이터(KaggleHub 'E-commerce Sales Data 2024')의 결측치·이상치를 체계적으로 처리하고, 가우시안 노이즈 기반 데이터 증강을 통해 추천 시스템 학습에 적합한 고품질 데이터셋을 구축한다. 이를 통해 데이터 희소성 문제를 완화하고, 모델의 일반화 성능을 확보하기 위한 기반 데이터를 마련한다.

## 방법론

> **참조: Paper §3.2 데이터 전처리 (Data Preprocessing), §3.3 데이터 증강 (Data Augmentation)**

### 1. 결측치 처리 (Paper §3.2)

원시 데이터에는 다양한 형태의 결측치가 포함되어 있어, 변수 특성에 맞는 다단계 처리 전략을 적용하였다.

- **시계열 속성 보간**: `user_id`, `time_stamp` 등 시계열 속성에 대해 순서 기반 보간법(forward fill → backward fill)을 적용하여 시간적 연속성을 보존하였다.
- **상품 ID 결측 대체**: `product_id`가 누락된 경우 `'unknown_product'`로 대체하여 레코드 유실을 방지하였다.
- **상호작용 유형 결측 대체**: `interaction_type`이 비어있는 경우 최빈값(mode)을 활용하여 결측을 보완하였다.

### 2. 중복 제거 (Paper §3.2)

완전히 동일한 행(row)을 제거하여 데이터 중복을 해소하였다.

### 3. 이상치 클리핑 (Paper §3.2)

수치형 변수에 대해 분위값 기반 클리핑을 적용하여 상하위 1%의 극단값을 제거하였다. 이를 통해 모델 학습 시 이상치로 인한 편향을 방지하고 안정성을 확보하였다.

### 4. 범주형 인코딩 및 수치형 정규화 (Paper §3.2)

- **범주형 변수**: `interaction_type` 등 범주형 변수에 Label Encoding을 적용하여 정수형으로 변환하였다.
- **수치형 변수**: 표준 정규화(Standardization)를 적용하여 평균 0, 표준편차 1의 분포로 정렬하고, 스케일 차이로 인한 영향을 제거하였다.

### 5. 시간 정보 처리 (Paper §3.2)

시계열 특성 강화를 위해 `timestamp` 변수로부터 연도(year), 월(month), 일(day) 파생 변수를 생성하여, 시점 기반 행동 패턴 분석 및 시즌성 평가에 활용할 수 있도록 구성하였다.

### 6. 가우시안 노이즈 기반 데이터 증강 — ×3배 (Paper §3.3)

실제 소비자 행동의 다양성을 반영하기 위해, 수치형 변수에 가우시안 노이즈를 주입하여 원본 데이터를 복제·변형한 2개의 증강 데이터셋을 추가 생성하였다. 이후 원본과 병합하여 최종 데이터셋을 구성하였다.

이 방식은 기존의 무작위 노이즈 삽입과 달리, 샘플 특성에 기반한 노이즈 주입이라는 점에서 원본 분포를 보존하면서도 현실적인 데이터 다양성 확보가 가능하며, 모델의 과적합 방지와 예측력 강화에 기여한다.

## 결과 요약

### 증강 전후 데이터 규모 비교

| 데이터 | 증강 전 (원본) | 증강 후 (×3배) | 비고 |
|--------|---------------|---------------|------|
| 고객 정보 | 3,900건 | **11,700건** | 원본 + 가우시안 노이즈 변형 2개 |
| 상호작용 정보 | 3,294건 | **9,000건** | 원본 + 가우시안 노이즈 변형 2개 |
| 상품 정보 | 10,002건 | **30,006건** | 원본 + 가우시안 노이즈 변형 2개 |

가우시안 노이즈 기반 증강을 통해 전체 데이터가 약 3배로 확장되었으며, 이는 후속 단계(고객 프로파일링, 상품 프로파일링, 추천 엔진 학습)에서 모델의 학습 다양성 확보와 과적합 방지에 기여한다.

## 회고

### 왜 이 방법을 선택했는가

이커머스 데이터는 본질적으로 희소성(sparsity) 문제를 안고 있다. 사용자 3,900명, 상호작용 3,294건이라는 규모는 추천 시스템 학습에 충분하지 않았다. 가우시안 노이즈 기반 증강을 선택한 이유는, 단순 복제(oversampling)와 달리 원본 데이터의 통계적 분포를 보존하면서도 샘플 간 미세한 변동을 부여하여 다양성을 확보할 수 있기 때문이다. 또한 결측치 처리에서 순서 기반 보간법(forward fill, backward fill)을 적용한 것은, `user_id`와 `time_stamp` 같은 시계열 속성의 시간적 연속성을 최대한 보존하기 위한 선택이었다.

### 잘된 점

- **3배 증강의 효과**: 데이터 규모를 3배로 확장함으로써 후속 단계의 K-Means 클러스터링, HDBSCAN 클러스터링, LightGBM 재랭킹 등에서 모델 일반화 성능이 향상되었다.
- **순서 기반 보간의 적절성**: 시계열 특성을 가진 변수에 forward fill → backward fill을 적용하여, 단순 평균 대체보다 시간적 맥락을 잘 보존할 수 있었다.
- **분위값 기반 클리핑**: 상하위 1% 클리핑은 극단적 이상치를 제거하면서도 데이터의 자연스러운 분포를 크게 훼손하지 않는 균형 잡힌 접근이었다.

### 개선할 점

- **증강 비율 최적화 실험 부족**: ×3배라는 증강 비율은 경험적으로 설정한 것이며, ×2배, ×5배 등 다양한 비율에 대한 체계적인 비교 실험을 수행하지 못했다. 증강 비율에 따른 추천 성능 변화를 정량적으로 분석했다면 더 근거 있는 선택이 가능했을 것이다.
- **범주형 변수 증강 미적용**: 가우시안 노이즈는 수치형 변수에만 적용되었으며, `interaction_type`이나 `category` 같은 범주형 변수에 대한 증강 전략(예: SMOTE 변형, 범주 재샘플링)은 고려하지 못했다.

### 배운 점

- **데이터 품질이 모델 성능의 기반**: 아무리 정교한 추천 알고리즘을 설계하더라도, 입력 데이터의 품질이 낮으면 성능 향상에 한계가 있다. 전처리 단계에서의 꼼꼼한 결측치 처리와 이상치 보정이 전체 파이프라인의 안정성을 좌우한다는 것을 체감하였다.
- **전처리 파이프라인의 재현성 확보**: 환경변수(`FAST_MODE`, `AUG_COPIES`)를 통해 실행 모드를 분리하고, 동일한 스크립트로 빠른 테스트와 논문 재현을 모두 지원하는 구조가 협업과 검증에 큰 도움이 되었다. 재현 가능한 파이프라인 설계의 중요성을 다시 한번 확인하였다.
