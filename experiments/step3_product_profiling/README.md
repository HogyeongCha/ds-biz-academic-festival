# Step 3: 상품 프로파일링

## 목적

상품의 의미적 특성과 마케팅적 가치를 체계적으로 분석하여 추천 시스템의 콘텐츠 이해력을 극대화한다. 단순 카테고리 분류를 넘어 상품의 본질적 가치와 맥락적 의미를 포착하기 위해, 텍스트 임베딩 기반의 의미 벡터화 → 밀도 기반 클러스터링 → 키워드 추출 → LLM 속성 태깅으로 이어지는 다단계 파이프라인을 설계하였다. 이를 통해 상품의 '의미적 DNA'를 정량화하고, 콘텐츠 기반 필터링과 하이브리드 추천에서 핵심적인 역할을 수행하는 상품 프로파일을 구축한다.

## 방법론

> **참조: Paper §4.2 상품 프로파일링**

본 연구에서는 LLM 기반 의미 분석과 계층적 클러스터링을 결합한 상품 프로파일링 파이프라인을 제안한다. 해당 파이프라인은 기존의 단순한 카테고리 분류를 넘어서, 상품의 본질적 가치와 맥락적 의미를 효과적으로 포착할 수 있도록 총 6단계로 구성된다.

### 1단계: 상품 정보 통합 (Paper §4.2 — 상품 정보 통합)

개별 상품에 대해 제품명(`Product Name`), 카테고리(`Category`), 제품설명(`about_product`) 등의 정보를 `[SEP]` 구분자로 결합하여 통합 텍스트 코퍼스(`product_text_corpus`)를 생성하였다. 분산된 상품 정보를 하나의 통합된 텍스트로 변환함으로써 상품의 다차원적 특성을 보존하며, 결측치는 빈 문자열로 대체하여 벡터화 과정에서의 라벨링 정확도를 확보하였다.

### 2단계: Embedding (Paper §4.2 — Embedding 및 차원 축소)

생성된 `product_text_corpus`를 기반으로 **Sentence-BERT(`paraphrase-mpnet-base-v2`)**를 활용하여 각 상품을 768차원 벡터 공간 상의 점으로 임베딩하였다. 이는 상품 간 의미적 유사도를 수치적으로 비교 가능하게 만들며, 향후 클러스터링 및 추천에서 핵심적인 역할을 수행한다.

- SBERT를 사용할 수 없는 환경에서는 **TF-IDF fallback**을 지원한다.

### 3단계: 차원 축소 — PCA + UMAP (Paper §4.2 — 차원 축소)

고차원 벡터의 노이즈를 제거하고 클러스터링 성능을 향상시키기 위해 2단계 차원 축소를 순차적으로 수행하였다.

1. **PCA (768 → 40~50차원)**: 고차원과 저차원 공간 간의 거리 상관관계를 효과적으로 보존하며 전역적 구조 보존에 탁월한 PCA를 통해 1차 차원 축소를 진행하였다. 누적 설명 분산과 개별 설명 분산을 계산하여 정보를 가장 잘 보존하는 차원 수를 탐지한 결과, **누적 설명 분산 95.4%, 개별 설명 분산 0.91%**의 수치를 보이는 40차원을 선정하였다.
2. **UMAP (→ 10차원 또는 2D 시각화)**: PCA 단계를 거친 후 노이즈를 줄이고 임베딩 안정성을 개선하기 위해 UMAP을 적용하였다. 클러스터링에는 10차원을, 시각화에는 2D를 사용하였다.

### 4단계: 클러스터링 — HDBSCAN 그리드서치 (Paper §4.2 — 클러스터링)

다양한 밀도의 클러스터를 허용하며 클러스터 수에 대한 사전 가정 없이 임의의 형태를 가진 클러스터 발견에 용이한 **HDBSCAN** 클러스터링 기법을 채택하였다.

**대분류 클러스터링:**

`min_cluster_size = [50, 100, 150, 200]`, `min_samples = [50, 100]` 조합으로 그리드서치를 수행하고, 다음 4가지 지표로 종합 평가하였다:

- **실루엣 점수 (Silhouette Score)**: 클러스터 내 응집도와 클러스터 간 분리도
- **칼린스키-하라바즈 지수 (Calinski-Harabasz Index)**: 클러스터 간 분산 대비 클러스터 내 분산 비율
- **데이비스-볼딘 지수 (Davies-Bouldin Index)**: 클러스터 간 유사도 (낮을수록 좋음)
- **던 지수 (Dunn Index)**: 최소 클러스터 간 거리 대비 최대 클러스터 내 거리

최적 조합: `min_cluster_size=50, min_samples=50` → **166개 대분류 클러스터**

**소분류 클러스터링:**

대분류의 모든 클러스터에 대해 `min_cluster_size = [10, 15, 20, 30]`, `min_samples = [5, 10, 15]` 조합으로 추가 그리드서치를 수행하였다. 최적 조합: `min_cluster_size=10, min_samples=5` → **전체 1,374개 소분류 클러스터**

### 5단계: 키워드 추출 — c-TF-IDF (Paper §4.2 — TF-IDF 기반 키워드 추출)

각 클러스터의 의미적 특성을 파악하기 위해 **c-TF-IDF(class-based TF-IDF)** 기법을 활용하여, 클러스터별로 가장 중요도가 높은 단어 7개를 대표 키워드로 추출하였다. 이를 통해 각 클러스터가 어떤 상품군을 대표하는지 해석 가능한 형태로 요약할 수 있다.

### 6단계: 속성 태깅 — LLM 기반 (Paper §4.2 — LLM 기반 키워드 추출)

클러스터링에서 포착되지 못한 의미적 특성을 보완하기 위해 **LLM(Gemini 2.5 Flash)**을 통해 마케팅 속성을 자동 태깅하였다(논문 원본은 Gemini 2.0 Flash를 사용하였으나 서비스 종료로 인해 2.5 Flash로 대체). 클러스터별 대표 키워드와 상품 정보를 입력으로 제공하여, 마케팅적 관점에서의 속성(예: 가격대, 타겟 고객층, 사용 시나리오 등)을 추출한다.

- Gemini API가 없는 환경에서는 **로컬 fallback** 로직으로 대체된다.

## 실행 방법

### 파이프라인 전체 경로에서 실행

```bash
python pipeline/scripts/phase3_product_profiling.py
```

### Step 내 독립 실행

```bash
python code/phase3_product_profiling.py
```

### 환경변수 옵션

| 환경변수 | 설명 | 예시 |
|----------|------|------|
| `EMBEDDING_BACKEND` | 임베딩 백엔드 선택 (`sbert` 또는 `tfidf`) | `EMBEDDING_BACKEND=sbert` |
| `USE_GEMINI` | Gemini API 사용 여부 (1=사용, 0=로컬 fallback) | `USE_GEMINI=1` |
| `GEMINI_API_KEY` | Gemini API 키 | `GEMINI_API_KEY=your_key` |
| `FAST_MODE` | 빠른 실행 모드 (TF-IDF fallback 자동 적용) | `FAST_MODE=1` |
| `PAPER_REPRO` | 논문 재현 모드 (SBERT + Gemini 사용) | `PAPER_REPRO=1` |

```bash
# 논문 재현 실행 (SBERT + Gemini)
EMBEDDING_BACKEND=sbert USE_GEMINI=1 PAPER_REPRO=1 python code/phase3_product_profiling.py

# 빠른 테스트 실행 (TF-IDF + 로컬 fallback)
FAST_MODE=1 EMBEDDING_BACKEND=tfidf python code/phase3_product_profiling.py
```

> **참고**: 이 단계는 Step 1의 산출물(`combined_products.csv`)을 입력으로 사용한다. Step 1이 먼저 실행되어 해당 파일이 존재해야 한다.

## 회고

### 왜 이 방법을 선택했는가

기존의 단순 카테고리 분류는 "Electronics"와 "Smart Home Devices"처럼 의미적으로 유사하지만 카테고리가 다른 상품을 동일 그룹으로 묶지 못하는 한계가 있었다. 상품의 본질적 가치를 포착하려면 텍스트 수준에서의 의미적 유사도를 계산해야 했고, 이를 위해 SBERT 임베딩을 선택하였다. SBERT(`paraphrase-mpnet-base-v2`)는 문장 수준의 의미적 유사도를 효과적으로 포착하며, 상품 설명 텍스트의 맥락적 의미를 768차원 벡터로 압축할 수 있었다.

클러스터링 기법으로 HDBSCAN을 선택한 이유는 세 가지였다. 첫째, 클러스터 수를 사전에 지정할 필요가 없어 데이터가 스스로 군집 구조를 드러내도록 할 수 있었다. 둘째, 밀도 기반 접근이므로 임의의 형태를 가진 클러스터를 발견할 수 있었다. 셋째, 노이즈 포인트를 자연스럽게 처리하여 어떤 클러스터에도 속하지 않는 이상 상품을 분리할 수 있었다.

### 잘된 점

- **PCA + UMAP 2단계 차원 축소**: PCA로 전역적 구조를 보존하면서 노이즈를 제거하고, UMAP으로 지역적 구조를 보존하면서 시각화까지 동시에 달성하였다. 특히 PCA에서 누적 설명 분산 95.4%를 유지하면서 768차원을 40차원으로 축소한 것은 정보 손실을 최소화하면서도 클러스터링 성능을 크게 향상시켰다.
- **c-TF-IDF 기반 해석 가능한 키워드 추출**: 각 클러스터가 어떤 상품군을 대표하는지 사람이 읽을 수 있는 키워드로 요약함으로써, 블랙박스가 되기 쉬운 임베딩 기반 클러스터링에 해석 가능성을 부여하였다. 이는 마케팅 실무자가 클러스터 결과를 직관적으로 이해하고 활용할 수 있게 해주었다.
- **HDBSCAN 그리드서치의 체계적 접근**: 8가지 하이퍼파라미터 조합을 4가지 평가 지표로 종합 평가하여, 단일 지표에 의존하지 않는 객관적인 최적 조합 선택이 가능하였다.

### 개선할 점

- **SBERT 모델 선택의 다양성 부족**: `paraphrase-mpnet-base-v2` 단일 모델만 사용하였다. `all-MiniLM-L6-v2`나 도메인 특화 모델(예: e-commerce 학습 모델) 등 다양한 임베딩 모델을 비교 실험했다면, 상품 텍스트에 가장 적합한 임베딩 공간을 찾을 수 있었을 것이다.
- **소분류 클러스터링 품질 편차**: 대분류 클러스터별로 소분류 클러스터링의 품질이 균일하지 않았다. 상품 수가 적은 대분류 클러스터에서는 소분류가 의미 있는 세분화를 제공하지 못하는 경우가 있었으며, 클러스터별 적응적 하이퍼파라미터 조정이 필요하다는 점을 인식하였다.
- **LLM 속성 태깅의 일관성**: Gemini를 통한 속성 태깅은 프롬프트에 따라 결과의 일관성이 달라질 수 있었다. 태깅 스키마를 더 엄격하게 정의하거나, few-shot 예시를 체계적으로 설계했다면 품질을 더 높일 수 있었을 것이다.

### 배운 점

- **텍스트 임베딩 기반 상품 이해의 가능성**: 카테고리 메타데이터만으로는 포착할 수 없는 상품 간의 의미적 유사성을 SBERT 임베딩이 효과적으로 드러내주었다. "무선 이어폰"과 "블루투스 헤드셋"처럼 카테고리는 다르지만 본질적으로 유사한 상품들이 동일 클러스터로 묶이는 것을 확인하면서, 텍스트 임베딩이 상품 이해의 핵심 도구가 될 수 있음을 체감하였다.
- **클러스터링 품질 지표의 종합적 활용 중요성**: 실루엣 점수만으로는 최적 클러스터링을 판단하기 어렵다는 것을 배웠다. 칼린스키-하라바즈 지수는 클러스터 수가 적을수록 높아지는 경향이 있고, 데이비스-볼딘 지수와 던 지수는 서로 다른 관점에서 클러스터 품질을 평가한다. 여러 지표를 종합적으로 고려해야 비로소 데이터의 실제 구조를 반영하는 클러스터링 결과를 얻을 수 있다는 점을 확인하였다.
